# 1.JUC 
## 1.1 场景
JUC 是 JDK 提供的**高并发编程工具集**，用于多线程安全控制、同步协作与任务调度。

## 1.2 方案
基于 AQS 队列同步器实现，底层依赖 **CAS + park/unpark + CLH 等待队列**。
包含五大类：原子类、锁体系、同步器、线程池、并发集合。

## 1.3 实现
### 1.3.1 原子类
基于 Unsafe CAS 实现无锁原子操作。
常用：AtomicInteger、AtomicLong、AtomicReference、LongAdder。
核心：CAS 自旋更新，保证原子性。

### 1.3.2 锁体系（AQS 核心）
AQS 三要素：
- state：int 同步状态，CAS 修改
- CLH 队列：双向链表，存放等待线程
- park/unpark：线程阻塞与唤醒

1. ReentrantLock
独占可重入锁，默认非公平。
抢锁：CAS(state,0,1)，失败入队 park。
解锁：state 减到 0，unpark 队首线程。

2. ReentrantReadWriteLock
读写分离锁，state 分高低 16 位。
读锁共享，写锁独占；支持写降级，不支持读升级。

3. Condition
与 Lock 绑定，每个 Condition 拥有独立等待队列。
await：释放锁 → 入 Condition 队列 → park
signal：节点移到 AQS 队列 → unpark
用于精准唤醒，如生产者消费者。

### 1.3.3 同步器
1. CountDownLatch
减计数式同步器，await 等待计数到 0，用于等待一批任务完成。

2. CyclicBarrier
循环屏障，一组线程相互等待，达到数量后统一执行。

3. Semaphore
基于 AQS 共享模式，控制并发线程数，用于限流。

### 1.3.4 线程池
ThreadPoolExecutor 核心参数：
corePoolSize、maximumPoolSize、workQueue、threadFactory、handler。
执行流程：核心线程 → 队列 → 最大线程 → 拒绝策略。
生产使用有界队列，拒绝策略优先 CallerRunsPolicy。

### 1.3.5 并发集合
1. ConcurrentHashMap
JDK8：数组+链表/红黑树，CAS + 节点锁，高并发无全局锁。

2. CopyOnWriteArrayList
写时复制，读无锁，适合读多写少。

3. BlockingQueue
阻塞队列，用于生产者消费者模型，线程安全、阻塞等待。

## 1.4 实战
- 简单计数：用原子类，不加锁
- 独占资源：ReentrantLock，finally 必须 unlock
- 读多写少：读写锁
- 精准唤醒：Condition
- 限流：Semaphore
- 批量等待：CountDownLatch
- 异步任务：自定义线程池，禁用 Executors

# 2.底层原理
## 2.1 通用底层
所有 JUC 同步组件底层：
- CAS：原子修改状态
- park/unpark：线程阻塞唤醒
- AQS：CLH 队列管理等待线程

## 2.2 通用执行链路
获取资源 → CAS 成功 → 执行业务
→ CAS 失败 → 入 AQS 队列 → park
释放资源 → state 更新 → unpark 队首线程 → 重新 CAS

## 2.3 常见问题
- 死锁：锁顺序相反、未释放锁、线程互相阻塞
- 锁失效：未用同一锁、内部调用未走代理
- 安全问题：使用非线程安全集合、共享变量未原子化、未正确加锁
