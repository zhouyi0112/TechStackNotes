# 1.并发集合
## 1.1 场景
JUC 提供的高并发线程安全集合，用于多线程环境下安全存取数据。

## 1.2 方案
基于 CAS + 细粒度锁 + 写时复制 + 阻塞队列实现，保证线程安全与并发性能。

## 1.3 实现
### 1.3.1 ConcurrentHashMap
1. 结构：数组 + 链表 + 红黑树
2. 核心机制：
   - 查询无锁
   - 插入：目标位置为空则 CAS 插入
   - 哈希冲突：锁住链表/红黑树根节点，synchronized 保证安全
   - 链表长度≥8 转为红黑树，≤6 退化为链表
3. 特性：key、value 均不允许为 null，高并发、高性能。

### 1.3.2 CopyOnWriteArrayList
1. 写时复制：写操作复制新数组，修改完替换引用
2. 读无锁，写加锁保证安全
3. 弱一致性，适合读多写少场景

### 1.3.3 CopyOnWriteArraySet
底层基于 CopyOnWriteArrayList 实现，去重、线程安全、读无锁。

### 1.3.4 BlockingQueue
阻塞队列，用于生产者 - 消费者模型。
队空 take() 阻塞，队满 put() 阻塞。
常用：
- ArrayBlockingQueue：有界、数组实现
- LinkedBlockingQueue：可设置有界、链表实现
- SynchronousQueue：无存储、直接交付

## 1.4 实战
ConcurrentHashMap：高并发 KV 存储
CopyOnWriteArrayList：读多写少、允许弱一致
ArrayBlockingQueue：固定容量、生产稳定
BlockingQueue 必须指定容量，避免 OOM

# 2.底层原理
## 2.1 通用底层
线程安全依靠：CAS、synchronized 细粒度锁、Lock、Condition。

## 2.2 ConcurrentHashMap
- 查询：volatile 无锁读
- 插入：CAS 或锁头节点
- 扩容：多线程协助迁移，无全局停顿

## 2.3 CopyOnWriteArrayList
写加锁 → 复制新数组 → 写入 → 替换引用
读直接访问旧数组，无锁

## 2.4 BlockingQueue
- 锁：ReentrantLock
- 等待/唤醒：notEmpty、notFull 两个 Condition
- 底层：park / unpark

## 2.5 常见问题
- ConcurrentHashMap 不允许 null 值
- CopyOnWrite 不适合大量写、大数据量
- 无界队列会导致内存溢出
